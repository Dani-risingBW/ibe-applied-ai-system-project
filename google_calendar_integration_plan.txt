OVERVIEW
--------
PawPal+ is a full-stack university study-room booking application built with
Streamlit. Students can search for available rooms across multiple campus
libraries, check Google Calendar conflicts, and confirm reservations. The
system also includes an AI assistant for plain-English requests.

FEATURES
--------
- Multi-library support: LibCal API, web scraper, and mock adapters.
- Real-time availability: Live slots filtered by date, duration, seats,
  and amenities.
- Conflict detection: Room double-booking, operating hours, daily limits,
  and Google Calendar overlap checks.
- Google Calendar sync: OAuth 2.0 with booking add/remove synchronization.
- AI natural-language search: Claude-assisted form filling from user prompts.
- AI conflict suggestions: Alternative recommendations when slots are blocked.
- Persistent storage: SQLite backing store for restart-safe bookings.
- Dry-run mode: Validate bookings without submitting.

QUICK START
-----------
Prerequisites:
- Python 3.11 or later.
- Google Cloud project with Calendar API enabled and OAuth client secret file
  named client_secret_*.json in project root.
- Gemini API key for AI Search (optional for non-AI app features).

Setup:
1. Create and activate virtual environment:
   - python -m venv .venv
   - Windows: .venv\Scripts\activate
   - macOS/Linux: source .venv/bin/activate
2. Install dependencies:
   - pip install -r requirements.txt
3. Create environment file:
   - cp .env.example .env
4. Set at minimum:
   - GEMINI_API_KEY=...
5. Run app:
   - streamlit run app.py

Docker:
- Start: docker compose up --build
- Stop: docker compose down

TESTING
-------
- Run all tests: python -m pytest
- Verbose: python -m pytest -v
- Specific module example: python -m pytest tests/test_booking_engine.py
- README currently documents 195 tests.

PROJECT STRUCTURE HIGHLIGHTS
----------------------------
- app.py: Streamlit UI (9 phases)
- booking_engine.py: Domain models + BookingEngine + ConflictDetector
- persistence.py: SQLite BookingStore (Phase 8)
- ai_assistant.py: Gemini-powered search + conflict suggestions (Phase 7)
- google_calendar_integration.py: OAuth2 Google Calendar sync (Phase 4)
- adapters/base.py: Abstract adapter contract
- adapters/libcal.py: SpringShare LibCal REST adapter
- adapters/scraper.py: Generic HTML scraper adapter
- adapters/mock.py: In-memory adapter for local dev/tests
- libraries.json: Library adapter and rule configuration

DEVELOPMENT PHASES
------------------
Phase 1: Domain models, BookingEngine, scaffold UI
Phase 2: Pluggable adapter layer (LibCal, Scraper, Mock)
Phase 3: Conflict detection with dynamic operating hours
Phase 4: Google Calendar OAuth2 integration
Phase 5: Streamlit UI redesign (multi-step booking flow)
Phase 6: Guardrails and edge-case hardening
Phase 7: AI assistant (natural-language search with Claude)
Phase 8: SQLite persistence (bookings survive restarts)
Phase 9: Production readiness (Docker, README, startup validation)

GOOGLE CALENDAR SETUP
---------------------
1. Open Google Cloud Console -> APIs & Services -> Credentials.
2. Create OAuth 2.0 Client ID (Desktop app).
3. Download JSON and place in project root as client_secret_*.json.
4. In app sidebar, click "Connect Google Calendar" on first use.
5. OAuth token is stored in token.json for reuse.

ENVIRONMENT VARIABLES
---------------------
- GEMINI_API_KEY: Required for AI Search.
- FOUNDERS_CLIENT_ID: LibCal OAuth client ID for Founders Library.
- FOUNDERS_CLIENT_SECRET: LibCal OAuth client secret for Founders Library.
- LAW_CLIENT_ID: LibCal OAuth client ID for Law School Library.
- LAW_CLIENT_SECRET: LibCal OAuth client secret for Law School Library.
- SCIENCE_CLIENT_ID: LibCal OAuth client ID for Science Library.
- SCIENCE_CLIENT_SECRET: LibCal OAuth client secret for Science Library.
